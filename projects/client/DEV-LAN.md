# 临时 LAN 协同开发指南

> ⚠️ **本功能仅在小组协同开发阶段临时使用，开发完成后务必关闭！**

---

## 快速启动 LAN 共享模式

1. 确保 `metro.config.js` 中的 **TODO(临时协作开发)** 代码块已启用（`server.host: '0.0.0.0'`）。
2. 执行以下命令启动：

```bash
npx expo start --web --lan
# 或
npm run start:lan
```

3. 在控制台输出的二维码下方，找到本机的 **LAN 地址**，例如：

```
› Metro waiting on exp://192.168.31.100:8083
› Web is waiting on http://192.168.31.100:8083
```

4. 告诉同组人员用浏览器打开该地址（如 `http://192.168.31.100:8083`）即可看到页面。

---

## ⚠️ 重要限制（必读）

| 场景 | 是否支持 |
|------|----------|
| 同局域网电脑查看页面 UI | ✅ 支持 |
| 同局域网电脑测试金蝶 API 登录/拉单 | ❌ **不支持**（浏览器 CORS 限制） |
| 真机/模拟器运行 App 测试金蝶 API | ✅ 支持（React Native 无 CORS） |

**原因说明**：
LAN 共享只是让 Metro bundler 监听 `0.0.0.0`，使其他电脑能加载到本地打包的 JS/CSS/HTML。
但页面中的 `fetch` 请求是**浏览器直接发往金蝶服务器**（`https://121.37.216.69`），浏览器同源策略（CORS）会阻止跨域请求，因此：

- **Web 预览共享** = 只能看 UI、填表单，无法真正登录或拉取检验单。
- **真机运行** = React Native 的 `fetch` 不受 CORS 限制，可以正常调用金蝶 API。

如需在 LAN 电脑上完整测试 API，请：
1. 使用 **React Native 真机/模拟器**运行；或
2. 在 LAN 电脑上搭建一个**同域代理**（如 nginx、vite 代理）转发请求到金蝶服务器；或
3. 临时禁用浏览器 CORS（仅限开发调试，不推荐长期用）。

---

## 开发完成后如何关闭

请按以下顺序清理，恢复安全状态：

1. **停止 LAN 启动命令**：在终端按 `Ctrl + C` 关闭 Expo 服务。
2. **删除/注释 `metro.config.js` 中的 TODO 块**：
   ```js
   /* TODO(临时协作开发 / LAN共享模式) ... 一直到 ... TODO END */
   ```
   将整个块注释掉或删除，恢复为默认仅本地访问。
3. **删除本文件 `DEV-LAN.md`**（可选，如不再需要）。
4. **检查 `package.json`**：如不再需要 `dev:share`、`start:lan` 等脚本，可一并移除。
